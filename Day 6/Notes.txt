Memory Allocation in Java
в”‚
в”њв”Ђ Two Main Areas
в”‚    в”њв”Ђ Stack Memory
в”‚    в””в”Ђ Heap Memory
в”‚
в”њв”Ђ Stack Memory
в”‚    в”њв”Ђ Used for local variables, method parameters, and references. :contentReference[oaicite:5]{index=5}
в”‚    в”њв”Ђ Managed automatically when methods are called. :contentReference[oaicite:6]{index=6}
в”‚    в”њв”Ђ Faster than heap memory. :contentReference[oaicite:7]{index=7}

в”‚
в”њв”Ђ Heap Memory

в”њв”Ђ Stack vs Heap (Key Differences)

в”‚    в”њв”Ђ Stack memory is automatically freed when a method ends. :contentReference[oaicite:14]{index=14}
в”‚    в””в”Ђ Heap memory is freed when objects are no longer used (GC). :contentReference[oaicite:15]{index=15}

в””в”Ђ Object Creation Memory Flow

Object Creation Flow

в”њв”Ђ 1. Class Loader Loads the Class
в”‚     в”њв”Ђ JVM checks if the classвЂ™s bytecode is already loaded
в”‚     в”њв”Ђ If not, ClassLoader loads the class *into JVM memory*

в”‚
в”њв”Ђ 2. Memory Allocation for Object (`new`)

в”‚     в””в”Ђ Then control returns back to calling method
в”‚
в””в”Ђ 4. Reference in Stack
      в”њв”Ђ The reference variable (like `dpRef`) that points to the object
      в”‚     is stored in **stack memory** :contentReference[oaicite:5]{index=5}
      в””в”Ђ So:
